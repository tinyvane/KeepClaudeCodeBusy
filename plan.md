# Keep Claude Code Busy - Claude 监控工具 - 项目规划

## 项目概述
Python 3.7+ + Tkinter + PyAutoGUI + OpenCV 构建的 Windows 屏幕监控工具，检测 Claude Code 输出区域画面变化，无变化时自动发送指令保持其持续工作，支持可视化区域选择和实时监控。

---

## 2026-03-03 Claude 建议

### 现状评估
解决了 Claude Code 在长任务中停止响应的实际痛点。OpenCV 图像对比检测是聪明的实现方式。随着 Claude Code 功能不断迭代，此工具的必要性可能会降低，但短期内仍有价值。

### 功能建议

#### 短期（稳定性提升）
1. **检测准确性**
   - 可调节灵敏度（避免微小变化误触发）
   - 排除鼠标光标影响（鼠标在区域内移动不算变化）
   - 截图对比算法优化（感知哈希替代像素对比）
   - 多区域同时监控

2. **触发策略**
   - 发送内容自定义（不仅是固定指令，支持列表随机选取）
   - 触发前等待时间配置（N秒无变化才触发）
   - 触发间隔限制（防止无限循环触发）
   - 工作时间限制（9-18点才运行）

3. **状态管理**
   - 系统托盘图标（最小化运行）
   - 运行日志（记录触发次数和时间）
   - 统计：今日触发次数、监控时长

#### 中期
1. **智能化**
   - 识别 Claude 具体停止原因（权限请求/错误/完成）
   - 根据不同情况发送不同指令
   - 错误检测（识别红色错误信息）

2. **多窗口支持**
   - 同时监控多个 Claude Code 实例
   - 窗口自动检测（不需要手动框选区域）
   - 多实例状态汇总视图

3. **通知**
   - 任务完成时通知（Claude 输出包含完成信号）
   - 长时间无进展时提醒用户
   - 钉钉/微信通知推送

#### 长期
1. **平台扩展**
   - 支持其他 AI 工具（GitHub Copilot、Cursor 等）
   - 通用"保持 AI 工具活跃"框架

2. **更优雅的方案**
   - Claude Code 官方可能添加自动继续功能（跟踪更新）
   - 直接集成 Claude Code MCP 钩子（官方 API）
   - 关注 Claude Code Hooks 功能（已在项目中用到）

3. **开源发布**
   - 对所有 Claude 重度用户有价值
   - GitHub 发布，配合详细文档

### 技术建议
- OpenCV 可替换为 `pillow + imagehash`（更轻量）
- 注意 Claude Code 版本更新后区域位置可能变化（需要重新选择区域）
- 打包为 `.exe`（PyInstaller --onefile）方便分发
- 与 `KeepClaudeCodeBusy` 项目合并（功能高度重叠）
