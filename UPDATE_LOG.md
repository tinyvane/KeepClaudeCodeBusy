# Claude Code 监控工具 - 更新日志

## 版本 1.0.0 (2025-10-14)

### 新增功能

#### 1. 记忆位置设置复选框
- **位置**: 参数设置区域
- **功能**: 用户可以选择是否在下次启动时自动恢复区域和点击位置
- **默认值**: 勾选（启用记忆功能）
- **说明**:
  - 勾选时：下次启动自动显示红色边框和之前选择的区域
  - 取消勾选：下次启动需要重新选择区域
  - 其他参数（检查间隔、触发时间、阈值、消息）始终保留

#### 2. 复选框状态实时反馈
- 勾选/取消勾选时在状态栏显示提示信息
- 绿色提示：已启用，下次启动将自动恢复
- 橙色提示：已禁用，下次启动需要重新选择

#### 3. 清除配置按钮
- **位置**: 区域选择区域，"选择区域"按钮旁边
- **功能**: 一键清除所有保存的配置和设置
- **清除内容**:
  - 选择的区域和点击位置
  - 屏幕上的红色边框
  - 配置文件 (monitor_config.json)
  - 所有参数恢复为默认值
- **安全性**: 需要确认对话框，防止误操作

#### 4. 配置文件版本管理
- 配置文件新增字段:
  - `version`: 记录程序版本号 (1.0.0)
  - `last_saved`: 记录最后保存时间
- 加载配置时显示版本信息和保存时间
- 便于未来版本升级和兼容性管理

#### 5. 区域选择成功反馈增强
- **视觉反馈**:
  - 区域信息文字加粗显示（绿色）
  - 弹出详细信息对话框，显示：
    * 监控区域的位置和大小
    * 点击位置的坐标
    * 红色边框提示
- **声音反馈**:
  - 选择成功时播放 Windows 系统提示音
  - 让用户明确知道操作已完成

### 改进内容

#### 1. 窗口尺寸调整
- 从 600x650 增加到 600x680
- 确保新增的复选框完整显示

#### 2. 按钮布局优化
- "选择区域"和"清除配置"按钮并排显示
- 使用容器框架统一管理

#### 3. 配置加载逻辑优化
- 先读取 `remember_position` 设置
- 仅在启用记忆功能时恢复区域设置
- 其他参数始终恢复，不受复选框影响

### 文件改动

- `monitor_tool.py`:
  - 新增版本号常量 `VERSION = "1.0.0"`
  - 导入 `datetime` 模块
  - 新增 `on_remember_position_changed()` 方法
  - 新增 `clear_config()` 方法
  - 优化 `save_config()` 添加版本和时间戳
  - 优化 `load_config()` 添加版本信息输出
  - 增强 `_update_region_result()` 添加声音和详细提示
  - 调整 GUI 布局（窗口高度、按钮排列）

### 使用场景

**场景1: 固定位置监控（默认）**
- 用户选择好区域后，保持复选框勾选
- 下次启动程序自动恢复上次的监控区域
- 适合长期监控同一个窗口位置

**场景2: 灵活位置监控**
- 取消勾选"记忆位置"复选框
- 每次启动需要重新选择监控区域
- 适合监控不同窗口或位置经常变化的情况

**场景3: 重置所有设置**
- 点击"清除配置"按钮
- 一键恢复默认状态
- 适合重新配置或解决配置问题

### 向后兼容性

- 旧版本配置文件可以正常加载
- 缺失的 `remember_position` 字段默认为 `True`
- 缺失的 `version` 和 `last_saved` 显示为"未知"

### 已知限制

- 声音提示仅支持 Windows 系统
- 非 Windows 系统不会播放声音，但不影响功能使用
