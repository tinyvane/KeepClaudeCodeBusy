# Keep Claude Code Busy

> **[中文](README_CN.md) | [English](README.md) | [Français](README_FR.md) | 日本語 | [Español](README_ES.md)**

Claude Code を寝ている間も継続的に動作させるために設計された Windows 画面領域監視ツール。

![バージョン](https://img.shields.io/badge/バージョン-1.0.0-blue.svg)
![Python](https://img.shields.io/badge/python-3.7+-green.svg)
![プラットフォーム](https://img.shields.io/badge/プラットフォーム-Windows-lightgrey.svg)
![ライセンス](https://img.shields.io/badge/ライセンス-MIT-orange.svg)

## ✨ 機能

- ✅ **ビジュアル領域選択** - 監視領域を選択するために矩形を描画
- ✅ **リアルタイム監視** - 画像比較により画面の変化を検出
- ✅ **スマート検出** - 30-120秒間変化がない場合に自動トリガー
- ✅ **自動メッセージング** - Claude Code に自動的にコマンドを送信
- ✅ **設定の永続化** - 設定を記憶（オプション）
- ✅ **クリーンな終了** - 起床時にいつでも監視を停止
- ✅ **ワンクリックパッケージ** - スタンドアロン EXE ファイルの作成

## 🚀 クイックスタート

### インストール

1. **依存関係のインストール**:
```bash
install.bat
```
または手動で:
```bash
pip install -r requirements.txt
```

2. **プログラムの実行**:
```bash
python monitor_tool.py
```
または `run.bat` をダブルクリック

### EXE の作成（推奨）

```bash
build_onedir.bat
```

実行ファイルは `dist/monitor_tool/monitor_tool.exe` に生成されます

## 📖 使用方法

### ステップ 1: 監視領域の選択

1. "选择区域" (領域選択) ボタンをクリック
2. 画面が半透明になります
3. Claude Code の出力領域の周りに矩形を**ドラッグ**して描画
4. **Enter キーを押して**確認（ESC でキャンセル）

### ステップ 2: クリック位置の選択

1. 領域を確認後、画面は半透明のまま
2. Claude Code の入力フィールドの位置を**クリック**
3. **再度 Enter キーを押して**確認

### ステップ 3: パラメータの設定

- **チェック間隔**: 変化をチェックする頻度（10-60秒、デフォルト: 30秒）
- **トリガー時間**: 変化なしでトリガーするまでの時間（30-120秒、デフォルト: 45秒）
- **類似度しきい値**: 画像比較の厳密さ（0.90-0.99、デフォルト: 0.98）
- **位置を記憶**: 次回起動時に領域を自動復元（チェックボックス）
- **メッセージ**: トリガー時に送信するテキスト（中国語サポート）

### ステップ 4: 監視の開始

"开始监控" (監視開始) ボタンをクリックして寝ましょう！😴

### ステップ 5: 起床時の停止

"停止监控" (監視停止) ボタンをクリックするか、ウィンドウを閉じます。

## 🎯 仕組み

1. N 秒ごとに監視領域のスクリーンショットを撮影
2. 現在のスクリーンショットを前回のものと比較
3. 変化が検出された場合 → タイマーをリセット
4. M 秒間変化がない場合 → 自動的に:
   - 入力フィールドをクリック
   - Ctrl+V でメッセージを貼り付け（中国語サポート）
   - Enter を押して送信
   - 監視を継続

## ⚙️ 設定

設定は `monitor_config.json` に保存されます:
- 選択された領域の座標
- クリック位置
- チェック間隔
- トリガー期間
- 類似度しきい値
- 位置記憶設定
- カスタムメッセージ

## 💡 ヒントとベストプラクティス

### 領域選択
- Claude Code の主要な出力領域を選択
- 点滅カーソルやタイムスタンプのある領域は避ける
- 領域は十分な大きさにする（少なくとも 100x100 ピクセル）

### パラメータの調整
- **敏感すぎる?** → しきい値を 0.99 に上げるか、トリガー時間を増やす
- **トリガーしない?** → しきい値を 0.95 に下げるか、トリガー時間を減らす
- **長いタスクの場合** → トリガー時間を 60-90 秒に設定
- **素早い応答の場合** → トリガー時間を 30-45 秒に設定

## 🐛 トラブルシューティング

### 問題: プログラムが Claude Code ウィンドウを見つけられない
**解決策**: ウィンドウが開いており、最小化されていないことを確認してください。

### 問題: 自動化が動作しない
**解決策**:
- マウスを画面の隅に移動しない（PyAutoGUI の安全機能）
- 必要に応じて管理者として実行
- 他のウィンドウが Claude Code を覆っていないことを確認

### 問題: 常にトリガーする（誤検出）
**解決策**:
- トリガー時間を増やす
- 類似度しきい値を上げる（0.99）
- 動的要素のない領域を再選択

### 問題: トリガーしない（検出漏れ）
**解決策**:
- しきい値を下げる（0.95）
- 領域が正しく選択されているか確認
- 監視が実行中であることを確認（ステータスをチェック）
